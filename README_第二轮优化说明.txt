第二轮优化说明（v2）

本次在第一版基础上继续做了第二轮优化，重点是“更像一个能推动询盘的网站”，而不仅是页面更好看。

一、本轮已完成的优化
1. 首页继续强化
- 明确网站的使用路径：浏览产品 → 了解流程 → 发起询盘
- 增加“买家第一次应提供什么信息”模块
- 增加 FAQ 预览
- 增加更清晰的分类入口和 CTA

2. 产品页继续强化
- 保留全部产品分组
- 每个产品卡片新增两个动作入口：
  - Request quote
  - Ask about sample
- 从产品页点击后，会自动把 category 和 style reference 带入 Contact 页面

3. Contact 页面继续强化
- 新增 Style reference 字段
- 新增 Requested shipping basis 字段
- 新增 Request type 字段
- 支持从产品页自动预填 category / style reference / message
- 增加“第一次询盘应包含哪些信息”的侧边说明

4. 新增 FAQ 页面
- 单独新增 faq.html
- 解决报价、运输条款、打样、联系方法等常见问题
- 已加入 FAQPage 结构化数据

5. SEO 与结构继续优化
- 所有核心页面增加更完整的 meta 描述与 OG 信息
- 内链关系更完整
- 页脚增加 FAQ 链接
- 页面层级与用途更清晰

6. JS 与交互优化
- FAQ 折叠交互
- Contact 页面 URL 参数预填
- 产品页到询盘页的路径更顺

二、这次优化特别有价值的点
按先后顺序看，最重要的是：
1. 买家从“看到产品”到“发起询盘”的路径更短
2. 询盘内容更完整，减少无效来回沟通
3. 站内页面之间开始形成更清晰的业务逻辑

三、仍然需要你们后续手动处理的事项
1. 必须把 contact.html 里的 Formspree 占位地址替换为真实表单 ID
2. 上线后必须逐页测试：
- 菜单跳转
- 产品页按钮跳转
- Contact 自动预填
- 表单真实送达
- 手机端显示
3. 如果你们有正式域名，建议后续继续补：
- canonical
- sitemap.xml
- robots.txt
- Google Search Console 提交
4. 如果要继续做第三轮，最值得补充的是：
- 工厂/车间真实照片
- 认证文件
- 每个产品类别的参数说明
- MOQ / 面料 / 尺码 / 包装 / 打样周期等真实业务数据

四、客观边界
本次仍然是静态前端优化，不涉及服务器端、数据库和后台系统改造。
没有虚构未证实的客户背书、认证、产能、交期等信息。
